# 文档生成提示词

## 任务

根据表结构设计结果，按照 PM3.0 标准模板生成数据库设计文档。

## 输入

- 表结构设计结果（JSON格式）
- 模块名称
- 冲突检测报告

## 输出格式

按照 `02_db_design_template.md` 格式生成 Markdown 文档，包含以下章节：

### 文档结构

```markdown
# XXX模块数据库设计文档

> 文档版本：v1.0
> 创建日期：YYYY-MM-DD
> 创建人：Claude
> 最后更新：YYYY-MM-DD

---

## 1. 概述

### 1.1 文档说明
### 1.2 数据库环境
### 1.3 设计原则

## 2. 命名规范

### 2.1 表命名规范
### 2.2 字段命名规范
### 2.3 索引命名规范

## 3. 字段类型规范

### 3.1 常用字段类型
### 3.2 长度设置建议

## 4. 表结构设计

### 4.x 表名：xxx（表说明）

#### 表说明
#### 表结构
#### 索引设计
#### 建表SQL

## 5. 表清单

## 6. 数据字典

### 6.x 枚举值定义
```

## 各章节生成规则

### 第1章 - 概述

固定内容：

```markdown
## 1. 概述

### 1.1 文档说明

本文档描述{模块名称}的MySQL数据库表结构设计，包括表字段定义、索引设计和命名规范。
文档基于需求规格说明书自动生成，遵循PM3.0数据库设计规范。

### 1.2 数据库环境

- **数据库类型**：MySQL 5.7+
- **字符集**：utf8mb4
- **排序规则**：utf8mb4_general_ci
- **存储引擎**：InnoDB

### 1.3 设计原则

- 所有表必须包含审计字段（id、create_by、update_by、create_time、update_time、is_del、remark）
- 使用软删除机制（is_del标记）
- 主键统一使用VARCHAR(32)存储UUID
- 合理设置字段长度和默认值
- 为高频查询字段建立索引
- 复用系统统一表（idm_*, permission_*, mam_*, mdm_*等），不新建此类表
```

### 第2章 - 命名规范

引用 db_design_rules.md 的命名规范部分。

### 第3章 - 字段类型规范

引用 db_design_rules.md 的字段类型规范部分。

### 第4章 - 表结构设计

为每个表生成以下小节：

#### 4.x 表名：{table_name}（{table_comment}）

##### 表说明

根据 entity_source 和字段注释生成表说明。

##### 表结构

生成字段表格：

```markdown
| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 备注 |
|--------|------|------|----------|--------|------|
| id | VARCHAR | 32 | 否 | - | 主键 |
| clue_id | VARCHAR | 16 | 否 | - | 线索ID |
| ... | ... | ... | ... | ... | ... |
```

##### 索引设计

生成索引表格：

```markdown
| 索引名 | 索引类型 | 字段 | 说明 |
|--------|---------|------|------|
| PRIMARY | 主键索引 | id | 主键 |
| uk_pm_clue_clue_id | 唯一索引 | clue_id | 线索ID唯一 |
| idx_pm_clue_is_del | 普通索引 | is_del | 软删除查询 |
```

##### 建表SQL

生成完整的 CREATE TABLE 语句：

```sql
CREATE TABLE `pm_clue` (
  `id` VARCHAR(32) NOT NULL COMMENT '主键ID',
  `clue_id` VARCHAR(16) NOT NULL COMMENT '线索ID',
  ...
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_pm_clue_clue_id` (`clue_id`),
  KEY `idx_pm_clue_is_del` (`is_del`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='线索主表';
```

### 第6章 - 数据字典

收集所有枚举类型的字段，生成枚举值定义：

```markdown
## 6. 数据字典

### 6.1 通用枚举值定义

#### is_del - 删除标记
| 值 | 名称 | 说明 |
|----|------|------|
| 0 | 未删除 | 正常数据 |
| 1 | 已删除 | 已软删除 |

### 6.2 业务枚举值定义

#### clue_status - 线索状态
| 值 | 名称 | 说明 |
|----|------|------|
| 0 | 不通过 | 未经审批或审批不通过 |
| 1 | 通过 | 审批通过 |
| 2 | 放弃 | 已放弃 |
```

## 冲突报告

在文档开头添加冲突报告（如有）：

```markdown
> [!WARNING]
> ## 统一表冲突检测报告
>
> 以下实体与系统统一表冲突，建议复用已有表结构：
>
> | 实体名称 | 冲突表名 | 建议处理方式 |
> |---------|---------|-------------|
> | 用户数据项 | idm_user | 复用idm_user表，通过逻辑外键关联 |
> | 部门数据项 | mdm_dept | 复用mdm_dept表 |
```

## 生成步骤

1. **准备数据**：整理表结构、索引、枚举值
2. **生成章节1-3**：概述和通用规范（固定内容）
3. **生成章节4**：逐个生成表结构设计
4. **生成章节5**：汇总表清单
5. **生成章节6**：汇总数据字典
6. **添加冲突报告**（如有）
7. **输出完整文档**

## 格式要求

1. 使用标准 Markdown 格式
2. 表格使用 GFM 格式
3. SQL 代码块使用 ```sql 标记
4. 所有注释使用中文
5. 保持章节编号连续
