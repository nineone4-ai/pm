# 实体提取提示词

## 任务

从需求规格说明书中提取所有数据实体定义。

## 输入

需求规格说明书（Markdown格式），包含以下章节结构：

```markdown
# 功能性需求
## 4.x. 模块名称
### 4.x.8. 数据项
#### XXX数据项
| 数据项名称 | 类型 | 长度 | 必填 | 备注 |
```

## 提取规则

### 1. 查找数据项章节

使用模式匹配查找所有四级标题（####）以"数据项"结尾的章节：

```regex
^####\s+(.+?数据项)\s*$
```

### 2. 解析数据项表格

每个数据项章节后应包含一个Markdown表格，格式如下：

```markdown
| 数据项名称 | 类型 | 长度 | 必填 | 备注 |
| --- | --- | --- | --- | --- |
| 字段1名称 | 文本 | 32字符 | 是 | 说明 |
| 字段2名称 | 数值 |  | 否 | 精度2位小数 |
```

### 3. 实体分类规则

根据数据项名称判断实体类型：

| 匹配模式 | 实体类型 | 说明 |
|---------|---------|------|
| `^(.+?)附件数据项$` | 附件实体 | 存储附件元数据 |
| `^(.+?)申请单数据项$` | 申请单实体 | 存储申请单信息 |
| `^(.+?)审批单数据项$` | 审批单实体 | 存储审批流程记录 |
| `^(.+?)数据项$` | 主实体 | 存储核心业务数据 |
| `^(.+?)线索数据项$` | 关联实体 | 存储关联关系 |

### 4. 字段解析规则

解析每个字段的以下属性：

- **原始名称**：数据项名称列的原始值
- **字段类型**：类型列的值（文本/数值/日期/时间型/布尔）
- **长度**：长度列的值（N字符/N汉字/空）
- **必填**：必填列的值（是/否）
- **备注**：备注列的值

### 5. 特殊字段识别

从备注中识别以下信息：

- **主键**：备注包含"主键"
- **外键**：备注包含"引用"、"关联"、"来源于"
- **唯一约束**：备注包含"唯一"
- **默认值**：从备注提取默认值信息
- **枚举值**：从备注提取取值范围（如：取值：A、B、C）
- **自动生成**：备注包含"自动生成"

## 输出格式

提取的实体应以JSON格式表示：

```json
{
  "entities": [
    {
      "entity_name": "线索",
      "entity_type": "主实体",
      "data_item_name": "线索数据项",
      "module": "线索管理",
      "fields": [
        {
          "original_name": "线索ID",
          "field_name": "clue_id",
          "data_type": "文本",
          "length": "16字符",
          "required": true,
          "comment": "主键，自动生成",
          "is_primary_key": true,
          "is_foreign_key": false,
          "is_unique": true,
          "default_value": null,
          "enum_values": null
        }
      ]
    }
  ]
}
```

## 处理步骤

1. **读取文档**：读取需求规格说明书全文
2. **章节分割**：按模块（4.x.）分割文档
3. **提取数据项**：在每个模块的4.x.8.章节中提取数据项
4. **解析表格**：解析每个数据项的字段表格
5. **字段转换**：将中文字段名转换为snake_case英文名
6. **关系识别**：识别外键关系和实体关联
7. **输出结果**：返回结构化的实体列表

## 字段命名转换规则

中文字段名 → snake_case 英文名：

| 中文模式 | 转换规则 | 示例 |
|---------|---------|------|
| XXXID/编号 | xxx_id | 线索ID → clue_id |
| XXX名称 | xxx_name | 项目名称 → project_name |
| XXX状态 | xxx_status | 线索状态 → clue_status |
| XXX类型 | xxx_type | 项目类型 → project_type |
| XXX时间 | xxx_time | 创建时间 → create_time |
| XXX日期 | xxx_date | 获取日期 → obtain_date |
| XXX编码 | xxx_code | 分公司编码 → branch_code |
| 是否XXX | is_xxx | 是否长期有效 → is_long_term |

## 冲突预处理

提取过程中标记以下情况：

1. **重复实体名称**：同一模块内出现同名数据项
2. **字段类型不明确**：类型列非标准值（文本/数值/日期/时间型/布尔）
3. **必填字段冲突**：同一字段在不同数据项中必填属性不一致
4. **疑似系统表**：实体名称匹配 idm_*, permission_*, mam_*, mdm_* 前缀
