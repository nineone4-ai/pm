## 业务规则

### 规则1：同一项目申请限制
同一项目下，同一种申请类型，只能存在1个单据。
- FIRST：首次申请
- SECOND：首次通过后的追加申请
- RESTART：通过但超1年未开工的重新申请

### 规则2：草稿唯一性
同一项目下，无论申请类型，只能有一个草稿。发起申请时若已存在草稿，轻提示去编辑。

### 规则3：列表展示维度
列表以项目维度展示（一个项目一行），默认展示最近操作的10条。

### 规则4：申请类型自动判断
```
IF 该项目无历史通过版本 → FIRST
ELSE IF 该项目有已通过版本 AND 当前时间-通过时间 > 1年 → RESTART
ELSE → SECOND
```

### 规则5：字段联动规则
- 业态细分联动业态：
  - 光伏 → [平地/渔光/山地/分布式]
  - 风电 → [平原/山地/海上]
  - 储能 → [电化学/抽水蓄能/压缩空气]

- 容量字段条件显示：
  - 业态=光伏 → 显示"光伏直流容量(MWp)"，必填
  - 业态=风电 → 显示"风电容量(MW)"，必填
  - 业态=储能 → 显示"储能规模MW/MWh"，必填
