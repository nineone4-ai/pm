# PM3 Step 1: PRD 解析器

## 职责边界

**只做这些**:
- 从 PRD 切片中提取功能点的原始描述
- 识别字段清单（名称、类型、必填条件）
- 提取按钮列表和点击动作描述
- 提取状态及状态说明
- 提取业务规则原文（不解释、不翻译）
- 提取审批节点和审批人角色

**不做这些**（留给后续 skill）:
- 不推断 Java 类名、表名
- 不设计 API 路径
- 不生成 DDL
- 不推断权限标识
- 不设计异常码

---

## 执行步骤

### Step 0: PRD 切片（继承自 pm3-superpower-spec-generator 的 Step 0）

如果上下文中已有 PRD 切片，直接使用，跳过此步。
否则执行切片逻辑（见 pm3-superpower-spec-generator SKILL.md 的 Step 0）。

### Step 1: 定位功能点

在 PRD 切片中定位目标功能点的行范围：
- 从 `#### <功能点名称>` 开始
- 到下一个同级或更高级标题结束

### Step 2: 逐节提取

按以下固定结构提取，每一项**严格引用原文**，不改写：

#### 2.1 功能描述
- 直接复制功能点开头的描述段落

#### 2.2 字段清单
从"列表/详情"、"数据项说明"小节提取：
```
| 字段显示名 | 必填 | 类型提示 | 备注/原文说明 |
```
字段类型仅填写 PRD 原文中明确写出的类型，没有则留空（不推断）。

#### 2.3 按钮清单
从"按钮逻辑"小节提取：
```
| 按钮名 | 显示条件(原文) | 点击动作(原文) | 校验规则(原文) |
```

#### 2.4 状态清单
从"状态说明"小节提取：
```
| 状态名 | 状态描述(原文) | 后续操作 |
```

#### 2.5 业务规则
从"补充逻辑"、"功能描述"等处提取**所有明确的业务规则**，原文摘录：
- 规则1: [原文]
- 规则2: [原文]

#### 2.6 审批流程（如有）
- 审批节点顺序（原文）
- 各节点审批人角色（原文）
- 特殊节点操作（原文，如：成本人员可修改目标成本流程）

#### 2.7 外部依赖（如有）
- 依赖的其他流程（原文）
- 自动带出的数据来源（原文）
- 接口清单中提到的依赖服务

### Step 3: 输出解析报告

输出文件: `02-analyzeprd/{模块名}/{功能点名}/prd-parse-report.md`

## 质量检查

输出前自检：
- [ ] 所有字段均来自 PRD 原文，无自行推断

- [ ] 外部依赖完整（不遗漏 PRD 中提到的接口依赖）
